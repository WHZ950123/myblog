# setTimeout与promise执行顺序

setTimeout与promise执行顺序

setTimeout是宏任务，promise是微任务

JavaScript执行完微任务就会执行下一个宏任务

# 详细执行顺序

JavaScript会把代码先走一遍，把微任务加入到微任务队列，宏任务加入到宏任务队列，走完主线程后，询问微任务队列是否有微任务，若有，则依次取出执行完；再询问宏任务队列是否有宏任务，若有，则取出第一个执行。再询问微任务队列是否有微任务，若有，再依次取出执行完；再询问宏任务队列是否有宏任务，若有，则取出第一个执行。依次执行。

# 例子1
//第一步，加入一个宏任务到宏任务队列
setTimeout(function(){
  console.log('set1');
})

//第二步，new Promise()里的代码是同步的
var p = new Promise(function(resolve,reject){
  console.log('promise1');
  //第三步，加入到宏任务队列
  setTimeout(function(){
    console.log('set2');
  })
  resolve();
})

//第四步，加入一个微任务到微任务队列
p.then(function(){
  console.log('then1');
})

//第五步，加入到宏任务队列
setTimeout(function(){
  console.log('set3');
})

//第六步，同步
console.log(123);

执行结果如下：

promise1
123
then1
set1
set2
set3
# 例子2
//第一步，加入到宏任务队列
setTimeout(function(){
   console.log('set1');
})

//第二步，new Promise()里的代码是同步的
var p = new Promise(function(resolve,reject){
   //第三步，同步输出  
   console.log('promise1');
   //第四步，加入到宏任务队列
   setTimeout(function(){
     console.log('set2');
	 var p3 = new Promise(function(resolve,reject){
		resolve();
	 })
	 p3.then(function(){
	   console.log('then3');
	 })
   })
   resolve();
})

//第五步，加入到微任务队列
p.then(function(){
  console.log('then1');
})

var p2 = new Promise(function(resolve,reject){
  resolve();
})

//第六步，加入到微任务队列
p2.then(function(){
  console.log('then2');
})

//第七步，加入到宏任务队列
setTimeout(function(){
  console.log('set3');
})

//第八步，同步输出
console.log(123);

执行结果如下：

promise1
123
then1
then2
set1
set2
then3
set3
# 例子2详解：

同步执行完，微任务队列里有p1.then()和p2.then()，宏任务队列里有3个setTimeout()函数，首先执行完微任务队列，输出then1和then2，微任务队列为空了。然后执行宏任务队列里的第一个任务，输出set1，宏任务队列里剩下2个setTimeout()函数。再查看微任务队列，发现为空，则再执行宏任务队列里的第一个任务，输出set2，然后发现一个微任务 p3，将其加入到微任务队列，此时宏任务队列里剩下1个setTimeout()函数。再查看微任务队列，发现有一个p3，执行，输出then3，微任务队列为空。再执行宏任务队列里的第一个任务，输出set3。
